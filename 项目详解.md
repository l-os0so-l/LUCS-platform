## 一、总体框架：一个完整的Web应用长什么样？

### 1.1 类比理解

想象你训练了一个YOLO模型来做遥感图像检测。现在你想让别人也能使用它：
- **前端（Frontend）**：就是用户看到的"界面"——按钮、图片上传框、结果显示区域。类似于你平时用的网页版ChatGPT的界面。
- **后端（Backend）**：就是实际运行YOLO模型的"服务器"——接收图片、调用模型推理、返回结果。类似于你本地的Python脚本，但它24小时运行在一台远程机器上。
- **前后端通信**：前端和后端通过 **HTTP API**（类似于函数调用的网络版）交换数据。

### 1.2 技术栈一览

| 层级 | 技术/框架 | 作用 |
|------|----------|------|
| **前端** | Vue 3 | 构建用户界面的JavaScript框架 |

### 2.1 一次完整的"单图检测"流程

以用户点击"单图检测"并上传一张遥感图片为例，整个系统的数据流动如下：

```

  │                                │                                │
  │  ① 点击"单图检测"              │                                │
  │  → 选择图片文件                │                                │
  │ ─────────────────────────────>│                                │
  │                                │                                │
  │                                │ ② 创建 FormData 对象           │
  │                                │    将图片文件 + 模型名称打包     │
  │                                │                                │
  │                                │ ③ 调用 detectSingleImage()     │
  │                                │    发送 HTTP POST 请求到        │
  │                                │    /api/detection/single        │
  │                                │ ──────────────────────────────>│
  │                                │                                │
  │                                │                                │ ④ 接收上传文件
  │                                │                                │    → 保存到 uploads/
  │                                │                                │
  │                                │                                │ ⑤ 调用 YOLO 模型推理
  │                                │                                │    → 读取图片
  │                                │                                │    → predict() 预测

  │                                │                                │

  │                                │                                │    → 保存到 static/results/
  │                                │                                │
  │                                │                                │ ⑦ 构造 JSON 响应
  │                                │                                │    返回给前端
  │                                │<────────────────────────────── │
  │                                │                                │
  │                                │ ⑧ 解析响应数据                  │
  │                                │    → 更新原始图片预览            │
```

### 2.2 前后端通信协议：RESTful API

前后端约定好了一套"接口规范"：

| 接口路径 | 方法 | 功能 | 请求内容 | 响应内容 |
|---------|------|------|---------|---------|
| `/api/detection/single` | POST | 单图检测 | 图片文件 + 模型名称 | 检测框列表 + 结果图URL |
| `/api/detection/history` | GET | 获取历史记录 | 分页参数 | 历史记录列表 |
| `/api/detection/detail/{id}` | GET | 获取检测详情 | 检测ID | 单条检测详情 |
| `/api/targets/list` | GET | 获取目标类别 | 无 | 支持的检测目标列表 |
| `/api/test/connect` | GET | 连通性测试 | 无 | 简单文本响应 |
| `/health` | GET | 健康检查 | 无 | 服务状态 |

## 三、前端详解：用户看到的界面是怎么实现的？

### 3.1 前端技术核心概念

在开始看代码前，先理解几个Vue 3的核心概念（类比Python思维）：

| Vue概念 | 类比Python/AI | 说明 |
|--------|-------------|------|
| **组件（.vue文件）** | 类（Class） | 一个.vue文件 = 一个独立UI模块，包含模板(HTML)、逻辑(JS)、样式(CSS) |
| **`<template>`** | 类的`__str__`或展示层 | 定义UI长什么样 |
| **`<script setup>`** | 类的`__init__`和方法 | 定义数据、函数、生命周期 |
| **`<style scoped>`** | 类的专属属性 | 只影响当前组件的CSS |
| **响应式变量（ref/reactive）** | 可自动触发回调的变量 | 数据变了，界面自动更新 |
| **Router（路由）** | 条件分支/跳转逻辑 | 根据URL显示不同页面 |
| **生命周期钩子** | `__enter__`/`__exit__` | 组件创建、挂载、卸载时的回调 |

### 3.2 入口文件：`main.js`

这是前端程序的"`if __name__ == '__main__'`"：

```javascript
import { createApp } from 'vue'
import { createPinia } from 'pinia'
import App from './App.vue'
import router from './router/index.js'
import * as ElementPlusIconsVue from '@element-plus/icons-vue'

const app = createApp(App)

// 注册所有 Element Plus 图标（全局可用）
for (const [key, component] of Object.entries(ElementPlusIconsVue)) {
  app.component(key, component)
}

app.use(createPinia())   // 启用状态管理
app.use(router)          // 启用路由
app.mount('#app')        // 挂载到 index.html 的 <div id="app"> 上
```

**逻辑解析**：
1. `createApp(App)` —— 以`App.vue`为根组件创建Vue应用实例
2. `app.use(router)` —— 注入路由系统，让页面可以跳转
3. `app.mount('#app')` —— 把Vue应用渲染到DOM中的`#app`节点

### 3.3 根组件：`App.vue`

这是所有页面的"外壳"：

```vue
<template>
  <!-- 如果是登录/注册/忘记密码页面，直接显示，不用布局 -->
  <router-view v-if="isAuthPage" />
  
  <!-- 其他页面使用 MainLayout 布局（左侧边栏 + 顶部导航 + 内容区） -->
  <MainLayout v-else>
    <template #sidebar><Sidebar /></template>
    <template #header><Header /></template>
    <template #content><router-view /></template>
  </MainLayout>
</template>
```

**实现逻辑**：
- `<router-view />` 是一个"占位符"，Vue Router会根据当前URL自动把对应的页面组件塞到这里
- 通过`computed`计算`isAuthPage`，判断当前是否是认证页面，决定使用哪种布局
- `MainLayout` 使用了 Vue 的**插槽（slot）**机制：把`Sidebar`、`Header`和页面内容分别填入对应位置

### 3.4 路由配置：`src/router/index.js`

路由就是"URL → 页面组件"的映射表：

```javascript
const routes = [
  { path: "/", redirect: "/login" },           // 根路径重定向到登录页
  { path: "/login", component: () => import("../views/LoginPage.vue") },
  { path: "/detection", component: () => import("../views/DetectionPage.vue") },
  { path: "/history", component: () => import("../views/HistoryPage.vue") },
  // ... 其他页面
];
```

**关键逻辑 —— 路由守卫**：

```javascript
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem("token");
  const authPaths = ["/login", "/register", "/forgot-password"];

  if (authPaths.includes(to.path)) {
    next();  // 认证页面直接放行
  } else if (!token) {
    next("/login");  // 没有token（未登录）→ 强制跳登录页
  } else {
    next();  // 已登录 → 放行
  }
});
```

这就像在入口处设置一个`if`判断：检查用户是否携带了"通行证"（token），没有的话一律去登录。

### 3.5 HTTP请求封装：`src/utils/request.js`

这是前端与后端通信的"信使"，基于 **Axios** 库：

```javascript
const service = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || '/api',
  timeout: 30000  // 30秒超时
})

// 响应拦截器：统一处理错误
service.interceptors.response.use(
  response => response.data,  // 直接返回 data 部分
  error => {
    ElMessage.error('请求失败：' + error.message)  // 弹出错误提示
    return Promise.reject(error)
  }
)
```

**为什么需要封装？**
- 统一配置baseURL，不用每个请求都写完整地址
- 统一错误处理，一处修改，全局生效
- 可以统一添加token认证头、loading状态等

### 3.6 API接口定义：`src/api/detection.js`

这里把每个后端接口封装成一个JavaScript函数：

```javascript
export const detectSingleImage = (data) => {
  return request({
    url: '/detection/single',
    method: 'post',
    data,
    headers: {
      'Content-Type': 'multipart/form-data'  // 上传文件必须用 multipart
    }
  })
}
```

**设计模式**：这种分层封装使得业务代码和HTTP请求逻辑解耦。页面组件只需要调用`detectSingleImage(formData)`，不关心底层是怎么发请求的。

#### D. 核心检测逻辑

```javascript
const performSingleDetection = async (file) => {
  const loading = ElLoading.service({ ... });  // 显示"正在检测中..."遮罩
  
  try {
    const formData = new FormData();
    formData.append("file", file);              // ① 把文件放入FormData
    formData.append("model_name", selectedModel.value);
    
    originalImage.value = URL.createObjectURL(file);  // ② 生成本地预览URL
    
    const response = await detectSingleImage(formData);  // ③ 发送到后端
    
    if (response.success && response.data) {
      detectionResult.value = response.data;
      resultImage.value = "http://localhost:8000" + response.data.result_image_url;
      ElMessage.success("检测成功！");
    }
  } catch (error) {
    ElMessage.error("检测失败，请稍后重试");
  } finally {
    loading.close();  // 无论成功失败，关闭loading
  }
};
```

**关键点**：
- `FormData` 是浏览器提供的对象，专门用于构造 multipart/form-data 请求（支持文件上传）
- `URL.createObjectURL(file)` 在浏览器内存中创建一个临时URL，用于本地图片预览（不需要先上传到服务器）
- 结果图的URL拼接了完整后端地址，因为后端返回的是相对路径

## 四、后端详解：模型推理服务是怎么实现的？

### 4.1 后端技术核心概念

| FastAPI概念 | 类比 | 说明 |
|------------|------|------|
| **路由（Router）** | Flask的`@app.route` | 定义URL路径和对应的处理函数 |
| **依赖注入** | 函数的默认参数 | 自动把需要的对象传给处理函数 |
| **Pydantic模型** | Python的`dataclass`+校验器 | 定义请求/响应的数据结构，自动校验类型 |
| **UploadFile** | 文件对象 | FastAPI封装的文件上传类型 |

### 4.2 入口文件：`main.py`

```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from fastapi.staticfiles import StaticFiles

# 创建FastAPI应用实例
app = FastAPI(
    title=settings.APP_NAME,
    version=settings.APP_VERSION,
    description="遥感目标检测平台后端API"
)

# 配置跨域访问（CORS）
app.add_middleware(
    CORSMiddleware,
    allow_origins=settings.CORS_ORIGINS,  # 允许哪些前端域名访问
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# 挂载静态文件目录（让上传的图片和结果图可以通过URL直接访问）
app.mount("/static", StaticFiles(directory=settings.STATIC_DIR), name="static")

# 注册路由
app.include_router(detection_router, prefix="/api")
```

**CORS解释**：浏览器的安全策略禁止一个域名的网页随意请求另一个域名的接口。`CORSMiddleware`在后端响应头中添加`Access-Control-Allow-Origin`，告诉浏览器"我允许这些前端域名访问我"。

**静态文件挂载**：`app.mount("/static", ...)` 意味着所有以`/static`开头的URL请求，FastAPI会直接去`static/`目录下找对应的文件返回，而不经过Python代码处理。这类似于Nginx的静态文件服务。

```python
            if hasattr(settings, key):
                setattr(settings, key, type(getattr(settings, key))(value))
    return settings
```

### 4.6 业务逻辑层：`app/services/detection_service.py`

这是整个后端的"心脏"，负责调用YOLO模型：

```python
class DetectionService:
    def __init__(self):
        self.model = None
        self._load_model()       # 启动时加载模型
        self._init_class_names() # 初始化类别名称

    def _load_model(self):
        model_path = settings.YOLO_MODEL_PATH
        self.model = YOLO(model_path)  # Ultralytics加载.pt权重文件
```

**单图检测核心逻辑**：

```python
def detect_single_image(self, image_path: str, model_name: str = "pest-v1") -> DetectionResult:
    start_time = time.time()
    detection_id = str(uuid.uuid4())  # 生成唯一检测ID

    # ① 调用YOLO模型推理
    results = self.model.predict(
        source=image_path,
        conf=settings.CONFIDENCE_THRESHOLD,  # 置信度阈值：低于0.5的结果丢弃
        iou=settings.IOU_THRESHOLD,           # NMS的IOU阈值
        save=False                            # 不自动保存，我们手动保存
    )

    # ② 解析检测结果
    boxes = []
    for result in results:
        for box in result.boxes:
            x1, y1, x2, y2 = box.xyxy[0].tolist()  # 检测框坐标（左上角+右下角）
            confidence = float(box.conf[0])           # 置信度分数
            class_id = int(box.cls[0])                # 类别ID
            class_name = self.class_names.get(class_id, f"class_{class_id}")
            
            boxes.append(DetectionBox(x1=x1, y1=y1, x2=x2, y2=y2,
                                      confidence=confidence, class_id=class_id,
                                      class_name=class_name))

    # ③ 保存结果图（带检测框标注）
    result_filename = f"result_{uuid.uuid4().hex}.jpg"
    result_path = str(BASE_DIR / settings.RESULT_DIR / result_filename)
    results[0].save(filename=result_path)  # Ultralytics内置的save方法会画框

    # ④ 构造返回结果
    return DetectionResult(
        detection_id=detection_id,
        image_url=get_file_url(image_filename, settings.UPLOAD_DIR),
        result_image_url=get_file_url(result_filename, settings.RESULT_DIR),
        boxes=boxes,
        total_objects=len(boxes),
        detection_time=round(time.time() - start_time, 3),
        model_name=model_name,
        created_at=datetime.now()
    )
```

**关键设计**：
- `DetectionService`是**单例模式**（模块末尾 `detection_service = DetectionService()`），确保整个应用只加载一次模型，避免重复加载导致的内存浪费和启动延迟
- 模型在**应用启动时加载**（而非请求时加载），这样第一次请求的响应速度也能保证

### 4.7 工具函数：`app/utils/file_utils.py`

```python
async def save_upload_file(file: UploadFile, upload_dir: str = UPLOAD_DIR) -> str:
    os.makedirs(upload_dir, exist_ok=True)
    
    # 使用时间戳+原始文件名，避免文件名冲突
    timestamp = int(time.time())
    filename = f"{timestamp}_{file.filename}"
    file_path = os.path.join(upload_dir, filename)
    
    with open(file_path, "wb") as f:
        content = await file.read()  # 异步读取上传的文件内容
        f.write(content)
    
    return filename

def get_file_url(filename: str, dir_path: str) -> str:
    # 把本地路径转换为可通过HTTP访问的URL路径
    url_path = dir_path.replace("\\", "/").replace("static/", "/static/")
    return f"{url_path}/{filename}"
```
