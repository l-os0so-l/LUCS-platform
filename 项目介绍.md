# LUCS Platform — 土地利用分类智能检测平台

> 基于深度学习的遥感影像语义分割 Web 平台，支持单图/批量/文件夹/视频四种检测模式，集成 AI 智能问答助手。

---

## 一、项目概述

LUCS Platform（Land Use Classification System）是一款面向遥感影像的土地利用分类智能检测平台。用户上传遥感影像后，平台基于训练好的语义分割模型自动识别图像中的 7 种地物类型（背景、荒地、建筑、道路、水域、耕地、森林），并生成分割结果图、叠加图及像素统计报告。

**在线体验**：前端 `http://localhost:5173`，后端 `http://localhost:8000`

---

### 3.2 单图检测 ⭐（本人实现）

**单图检测是平台最核心的功能，由我负责完成前后端的完整实现。**

**实现内容：**

- **后端推理服务**：我独立编写了 `DetectionService` 单例类，负责模型加载、图像预处理、推理、后处理、可视化生成和像素统计计算。模型输入 512×512，输出 7 类 logits，经 argmax 后映射为伪彩色掩码。
- **前端展示页面**：我实现了 DetectionPage.vue 中的单图检测展示区域，采用三图并排对比布局（原图 + 分割结果 + 叠加图），标签置于图片下方避免遮挡。
- **像素统计**：我在后端计算各类别像素占比，前端以进度条形式可视化展示。
- **AI 诊断联动**：我在单图检测完成后，自动调用 AI 问答接口生成诊断建议（如"水域面积可观，注意水资源保护"）。
- **类别映射校正**：我人工确认并全局修正了 7 类映射关系（`0:背景, 1:荒地, 2:建筑, 3:道路, 4:水域, 5:耕地, 6:森林`），前后端保持一致。
- **用户绑定存储**：我将检测结果写入 SQLite 数据库并绑定当前用户 ID，实现了用户隔离的历史记录功能。

**检测流程：**

```
用户上传图片
    ↓
后端 DetectionService 加载 land_seg_best.pth
    ↓
预处理 → 512×512 → 归一化 → Tensor
    ↓
模型推理 → 7 类 logits → argmax → 掩码
    ↓
生成伪彩色分割图 + 叠加图 + 像素统计
    ↓
保存至数据库（绑定 user_id）
    ↓
前端三图对比展示 + AI 诊断建议
```

**性能表现**：单张 512×512 推理约 40ms（RTX 4060）。

### 3.3 批量检测

支持一次上传多张图片，循环执行单图检测逻辑，底部显示批量缩略图条，点击可切换查看各图结果。

---

### 3.4 文件夹检测

利用 `<input webkitdirectory>` 实现文件夹整体上传，后端自动过滤非图像文件，按顺序批量处理。我修复了 `webkitdirectory` 上传时文件路径包含子目录导致的 `FileNotFoundError`，通过 `Path(file.filename).name` 剥离路径。

---

### 3.5 视频检测

- 上传视频后，按设定的帧率（1~10 fps）抽取关键帧进行实时检测
- 左右对比布局：左侧原视频播放，右侧实时显示当前帧分割结果
- 右侧面板显示当前帧像素统计，检测设置和检测帧率面板默认折叠以腾出空间

---

### 3.6 历史记录

所有检测结果自动保存至 SQLite，按用户隔离。支持查看详情、删除记录。早期记录兼容 JSON 文件回退。

### 3.7 AI 智能问答 ⭐（本人实现）

**AI 问答模块由我独立完成后端接口和前端页面的设计与实现。**

**后端实现（`backend/app/api/qa.py`）：**

- 我设计并编写了对接 Moonshot API 的问答接口，采用 OpenAI-compatible SDK 调用 `moonshot-v1-8k` 模型。
- 我编写了详细的 **Grounded System Prompt**，将平台真实信息（模型架构、各类 IoU、mIoU 范围、支持功能）嵌入系统提示词中，确保 AI 回答严格基于事实，禁止编造或夸大指标。
- 我实现了 **超时保护**（15 秒单次超时）和 **自动重试机制**（最多 2 次，指数退避），大幅降低偶发服务器错误的影响。
- 我将 `max_tokens` 限制为 1024，避免超长回答拖慢响应。
- 我编写了完善的日志记录，每次调用记录模型名称、耗时和尝试次数，便于排查问题。

**前端实现（`frontend/src/views/QAPage.vue`）：**

- 我实现了完整的聊天界面，支持用户输入、快捷问题标签、Markdown 渲染、打字动画。
- 我添加了 **延迟友好提示**：当请求超过 8 秒未返回时，显示"正在努力思考中，请稍候…"，避免用户觉得页面卡死。
- 我处理了完整的错误边界，API 失败时友好提示用户。

**AI 问答的核心设计原则：**

| 原则 | 实现 |
|:---|:---|
| 如实说明 | 明确告知 mIoU 约 65%~70%，属于教学/演示级别 |
| 不夸大 | 禁止编造未实现的功能或更高的精度指标 |
| 坦诚边界 | 用户问及不支持的功能时，明确告知暂不支持 |
| 简洁友好 | 使用中文，回答精炼 |

## 四、模型训练与评估 ⭐（本人完成）

**本项目中的深度学习模型由我从零开始训练、调优和评估。**

### 4.1 数据集处理

我选用 **LoveDA** 遥感语义分割数据集，并独立完成了全部数据预处理流程：

- 合并原始 Train + Val 共 4188 张 1024×1024 图像
- 标签重映射：LoveDA 原始标签值不连续（含 0 和 1~7），我将其重映射为连续的 0~6
- Patch 切分：将大图切分为 512×512 patch（stride=256，50% 重叠）
- 背景过滤：过滤掉 >90% 背景的 patch，避免训练集被无效样本占据
- 最终获得约 28,000 张有效训练 patch
- 按 8:1:1 重新划分为 Train / Val / Test

### 4.2 模型架构选择

我对比了多种 Encoder + Decoder 组合，最终选定 **ResNet50 + DeepLabV3+**：

```
Input (512×512×3)
    │
    ▼
┌─────────────┐
│  ResNet50   │  ← ImageNet 预训练权重（Encoder）
└──────┬──────┘
       │
  ▼ ASPP 模块（多尺度空洞卷积：rate=1,6,12,18）
       │
┌─────────────┐
│ DeepLabV3+  │  ← 解码器，融合低层边界特征
└──────┬──────┘
       │
Output (512×512×7 classes)
```

**我的实验对比结果：**

| # | Encoder | Decoder | Val mIoU | 我的结论 |
|:---|:---|:---|:---|:---|
| 1 | ResNet18 | U-Net | 60.6% | 本地 baseline |
| 2 | ResNet50 | DeepLabV3+ | 67.7% | 初版（仅用 Train）|
| 3 | **ResNet50** | **DeepLabV3+** | **69.74%** | **最佳（Train+Val）✅** |
| 4 | ResNet101 | DeepLabV3+ | 69.18% | 更大模型反而更差 |
| 5 | ResNet101 | FPN | 69.31% | 略逊于 #3 |

**关键发现（由我得出）：**

1. **数据量 > 模型容量**：Train+Val（28K patch）比仅用 Train（17K）提升 2% mIoU
2. **ResNet50 是甜点**：ResNet101 参数量翻倍但表现更差，28K 数据不足以发挥其优势
3. **Decoder 比 Encoder 更重要**：ResNet18+U-Net 60.6% vs ResNet50+DeepLabV3+ 69.74%，Decoder 贡献 9%
4. **适度增强 > 重度增强**：GridDistortion、CoarseDropout 等强增强反而让遥感图像失真

### 4.3 训练策略（由我设计）

| 配置 | 我的设计 |
|:---|:---|
| Loss | Dice(0.5) + CrossEntropy(0.5) |
| 类别权重 | 平方根反比频率（避免纯逆频率的极端权重）|
| 优化器 | AdamW，Encoder LR=5e-5，Decoder LR=5e-4 |
| 学习率调度 | CosineAnnealingLR |
| Warmup | 前 2 epoch 冻结 Encoder，只训练 Decoder |
| 数据增强 | RandomCrop、Flip、Rotate90、BrightnessContrast |
| 混合精度 | AMP（速度 +30%，显存 -40%）|
| 早停 | patience=10，监控 val_mIoU |

### 4.4 评估结果

**最终模型验证指标（我测得）：**

| 类别 | IoU |
|:---|:---|
| 荒地 | 79.5% |
| 背景 | 77.2% |
| 耕地 | 74.4% |
| 水域 | 66.0% |
| 道路 | 62.9% |
| 建筑 | 61.8% |
| 森林 | 58.9% |
| **平均 mIoU** | **69.74%** |

### 4.5 推理性能（我测得）

| 输入尺寸 | 模式 | 速度 | 显存 |
|:---|:---|:---|:---|
| 512×512 | 标准 | ~40ms | ~2GB |
| 512×512 | TTA（4-flip）| ~150ms | ~3GB |
| 1024×1024 | 滑动窗口 | ~600ms | ~4GB |

## 五、快速启动

### 5.1 环境准备

```bash
# 1. 创建 conda 环境
conda create -n rsod-web python=3.10
conda activate rsod-web

# 2. 安装后端依赖
cd backend
pip install -r requirements.txt

# 3. 配置 API Key
echo "MOONSHOT_API_KEY=your_key_here" > .env

# 4. 安装前端依赖
cd ../frontend
npm install
```

### 5.2 一键启动

**Windows 批处理（我编写）**：
```bash
cd rsod-web-platform
start-all.bat
```

**Node.js 脚本（我编写）**：
```bash
cd frontend
node scripts/dev-all.js
```

**手动启动**：
```bash
# 终端 1：后端
cd backend
python -m uvicorn main:app --host 0.0.0.0 --port 8000

# 终端 2：前端
cd frontend
npm run dev
```
